Section 01

Ribbon overview

The "Elite Scan" tab with four panels: Analysis, Viewer, Overlay, and Point Clouds

Ribbon panels & buttons

Elite Scan

EliteScan installs a dedicated "Elite Scan" tab in the Revit ribbon. It contains four panels grouping point cloud analysis, visualization, and data management tools. Supports Revit 2022 through 2026.

Analysis panel

  • Place Scans — Auto-places scan locations from RCS origins
  • Run Analysis — CloudCompare deviation analysis workflow
  • View Report — Opens comprehensive results window
  • Settings — Configure scoring thresholds and categories

Viewer panel

  • 360 Viewer — Dockable equirectangular panorama pane

Overlay panel

  • Show Overlay — Colors elements by score (A-F grades)
  • Clear Overlay — Removes all color overrides

Point Clouds panel

  • Revit 2 ReCap — Shows point cloud transform data with per-value copy buttons for ReCap alignment
  • Cloud Reset — Resets RCP visibility to full density
  • Load Cloud — Batch-loads RCP files into project

Icons auto-swap between Light and Dark variants to match Revit's current UI theme. The add-in uses ricaun.Revit.UI for ribbon creation with automatic DPI scaling.

New in v1.3.0: deviation analysis is now occlusion-correct (each scan is compared against its own per-scan point cloud) and requires a one-time per-project calibration in Settings (see Section 03). The results window adds a Combine Analyses button for merging multiple per-area analyses into one report (see Section 12). No new ribbon buttons were added — both features live inside existing windows.

Analysis panel — Place Scans

Auto-places Scan Location family instances at each scan origin from RCS point cloud files

Place Scans button

Analysis > Place Scans

Automatically places Scan Location family instances at each scan origin detected in linked point cloud files. Each instance is populated with metadata required for analysis.

Requirements

  • Point cloud linked as RCP or RCS in the active project
  • Scan Location family loaded in the project (auto-prompts if missing)
  • RCS files accessible in the RCP's source folder structure
  • Active 3D view or plan view for placement visualization

What it sets

  • ScanID — Scan name from RCS file
  • PointCloudId — PointCloudInstance.Id for reference
  • Level — Closest Revit level by elevation
  • AnalysisRadius — Default 50 ft (configurable)
  • SupportFolderPath — Parent folder of RCS file
  • Location — XYZ coordinates of scan origin
  • Panorama orientation — Baked at Place time from the chosen mode (new in v1.3.0)

Skips scans that already have instances placed. If an instance's ScanID matches an existing instance's ScanID parameter, placement is bypassed to avoid duplicates.

Panorama orientation baked here (new in v1.3.0)

  • Mode chosen at Place time — FARO or E57, picked when you run Place Scans, not later in the 360 Viewer
  • Baked into each scan location — the correct viewing direction is stored on the family instance
  • Corrected formula — fixes the prior panorama misalignment (roughly 43° off in some projects)
  • To switch modes — re-run Place Scans for the project; the 360 Viewer simply trusts what was baked (see Section 08)

Section 03

Per-scan E57 setup & calibration

One-time per-project setup that makes deviation analysis occlusion-correct — required before Run Analysis in v1.3.0

Why per-scan E57 — occlusion-correct accuracy

Analysis > Settings

Through v1.2.9, deviation analysis compared every scan against the single merged point cloud (all scanner setups combined). That meant a scanner standing in one room still produced dense "deviation" on a beam above a ceiling or a wall it could never physically see — phantom results that made elements look worse than they are. As of v1.3.0, each scan is compared against its own point cloud only. A scanner only has points where it could actually see, so things it couldn't see simply have no points and no phantom deviation. This is a large accuracy improvement, but it requires a one-time setup per project, described on the next slides.

What this fixes

  • No phantom deviation on geometry a scanner couldn't see (beams behind walls, above ceilings, around corners)
  • True per-scan visibility — occlusion handled by physics, not guesswork
  • More trustworthy scores — elements graded only on points a scanner actually captured
  • Cleaner multi-scan picture — each scan contributes only what it really saw

What it requires of you

  • Split the project scan into one file per scan with the separate E57ImageExtractor tool (one time per project)
  • Associate & Calibrate in Settings so EliteScan can line up each scan with the Revit point cloud
  • Re-calibrate if you move the cloud — analysis stops if the saved calibration is stale

There is no fallback to the old merged-cloud method. If a point cloud has not been set up, Run Analysis stops with a clear message rather than producing less accurate results.

EliteScan Tutorial 5b

Step 1 — Split the scan with E57ImageExtractor

External Tool

E57ImageExtractor is a separate, standalone tool (run once per project, outside Revit). It takes the project's E57 export and splits it into one E57 file per individual scan, plus a small companion file that records each scan's origin. EliteScan reads these per-scan files during analysis.

What it producesDetails
_scans folder A folder named after the source E57 with "_scans" appended, containing one E57 per scan setup. This is the folder you point EliteScan at in Settings.
Companion sidecar file A small file written next to the per-scan E57s describing each scan's origin and point count. Calibration uses this to line scans up with Revit.
Integrity check The split is verified after it runs. Once verified, the large original project E57 can be deleted — only the _scans folder is needed going forward.

Run this once per project (or again if you re-export the scan data). EliteScan does not split the E57 itself — it consumes the output of this tool.

EliteScan Tutorial 5c

Step 2 — Associate & Calibrate in Settings

Analysis > Settings

Settings has a new section, "Point cloud scans (occlusion-correct)". It lists every point cloud linked into the project. For each one, point it at the matching _scans folder produced in Step 1, then click Calibrate. Calibration lines up the per-scan E57 origins with the point cloud as placed in Revit and saves the result so analysis is fast afterward.

ControlWhat it does
Point cloud list One row per linked point cloud, each showing its current setup status (see indicators below).
Browse / Associate Pick the _scans folder that belongs to that point cloud. The folder must contain the per-scan E57s and the companion file from Step 1.
Calibrate Aligns the scan origins to the Revit point cloud and caches the result. Run once per point cloud (and again if you move the cloud).
Calibration cache size Shows how much disk the saved calibrations occupy. Calibrations are small and stored per project.

Per-point-cloud status indicators

  • Not set up — no _scans folder associated yet; analysis of this cloud's scans will stop
  • Associated, not calibrated — folder linked but Calibrate has not been run yet
  • Calibrated — ready for analysis; shows when it was calibrated
  • Stale (cloud moved) — the point cloud was moved or replaced since calibrating; re-run Calibrate
EliteScan Tutorial 5d

What happens if calibration is missing or stale

Analysis > Run Analysis

When you click Run Analysis, EliteScan first checks that every point cloud the chosen scans belong to is calibrated and still current. If any check fails, the whole run stops before doing any work and tells you exactly which point cloud needs attention. There is deliberately no fallback to the old, less accurate merged-cloud method.

Setup required (run stops here)

  • E57ImageExtractor has been run for the project and produced a _scans folder
  • Each point cloud has its _scans folder associated in Settings
  • Calibrate has been clicked and succeeded for each point cloud
  • The point cloud has not moved in Revit since it was calibrated (otherwise re-Calibrate)

When the run stops, you'll see

  • A clear message naming the point cloud and the reason (not set up, can't read scans, or stale)
  • No partial results — nothing is analyzed until setup is complete
  • The fix — open Settings, associate the _scans folder and/or click Calibrate, then re-run

Individual scans with no matching per-scan E57 (or a name mismatch) are skipped and called out in the analysis summary rather than stopping the whole run.

EliteScan Tutorial 7

Run Analysis — Progress Dialog

Analysis > Run Analysis

First, a calibration gate runs once for the whole batch: EliteScan confirms every point cloud the chosen scans belong to is calibrated and current (see Section 03). If that check fails the run stops here. Otherwise a modal progress dialog displays the current scan, operation step, progress bar, and cancel button, and each scan runs through the steps below.

StepWhat happens
0. Calibration gate Runs once before the loop. Validates the per-scan E57 calibration for every point cloud the chosen scans belong to. Missing, unreadable, or stale (cloud moved) — the run stops with a clear message and nothing is analyzed. No fallback to the old merged-cloud method.
1. Export Geometry Collects elements within the scan's analysis radius. Filters by included categories (Settings). Exports the model surfaces to a mesh file used as the comparison reference.
2. Resolve per-scan cloud Finds the scan's own per-scan E57 (from the calibration set up in Section 03) instead of the merged point cloud. A scan with no matching per-scan E57 (or a name mismatch) is skipped and called out in the summary.
3. CloudCompare C2M CloudCompare applies the saved calibration so the scan lines up with the model, optionally crops to the analysis area for speed, then measures the distance from each point to the model surface.
4. Compute Statistics Parses CloudCompare output. Groups points by originating element using spatial queries. Computes mean deviation, max deviation, % in tolerance, point count per element. Assigns letter grades (A-F). Saves to the project's Results folder (see Section 13).

Canceling aborts the current scan and skips remaining scans. Partial results are not saved. CloudCompare executable path must be configured in Settings, and per-scan E57 calibration must be set up first (Section 03).

Results Window — Global Scan Selection Controls

Analysis > View Report

Four radio buttons control how element scores are aggregated when multiple scans cover the same element. Applies to the entire element list.

ModeHow it works
Best Visibility Uses the scan with the most points captured for this element. Prioritizes completeness of data over score. Default mode.
Best Score Uses the scan that gave the element its highest letter grade (e.g., if Scan A = B and Scan B = A, uses Scan B's score). Optimistic scoring.

Results Window — What's new in v1.3.0

Analysis > View Report

Several results-window changes land in v1.3.0: a live point-count filter, durable per-project review state, and a correction to how reviewed elements affect category accuracy.

ChangeWhat it means for you
Minimum-point filter (live) A toggle in the results window hides elements/scans at or below a point-count threshold (set in Settings). By default it only hides zero-point "No Data" rows so the list shows what was actually scanned. Toggling it recalculates the visible list immediately.

  • Flagged, never lost — if an element's geometry or score changed since you reviewed it, it shows a "data changed since review" note but keeps the approval
  • Conflicts surfaced — differing approvals for the same element become a conflict that counts as unreviewed until a person resolves it; nothing is silently discarded
  • First open migrates history — existing approvals from past results files are seeded into the project review store automatically

Section 06

Scoring System

Letter grade thresholds and multi-scan scoring modes

Scoring — Grade Thresholds

Settings

Elements receive letter grades (A-F) based on the percentage of captured points within the tolerance threshold (default 0.05 ft / 0.6 inches). Verified elements receive a "Reviewed" grade.

A — 90%+ in tolerance
B — 80%+ in tolerance
C — 70%+ in tolerance
D — 60%+ in tolerance
F — Below 60%
Reviewed — Verified by user

Thresholds are fully configurable in Settings. Tolerance defines the acceptable deviation distance (default 0.05 ft). Grade thresholds define the percentage cutoffs for A/B/C/D/F.

Multi-scan scoring modes

  • Best Visibility — Uses scan with most captured points. Default. Prioritizes data completeness.
  • Best Score — Uses scan with highest grade. Optimistic. Good for "prove it's right" scenarios.
  • Top 10 Average — Averages scores from top 10 scans by point count. Balanced. Reduces outlier influence.
  • Custom — Allows per-element scan selection. Maximum control. Use for investigating conflicts.

EliteScan Tutorial 18

Verification Screenshot Workflow

Results Window

Two methods for associating screenshots with verified elements: automated Capture workflow or manual Upload workflow.

Capture button workflow

  • 1. Click Capture in element row
  • 2. Panorama Viewer opens if not already visible
  • 3. Banner appears at top: "Navigate to view element. Press Capture when ready."
  • 4. Auto-zooms Revit viewport to element
  • 5. User navigates the 360° panorama to show element
  • 6. User clicks Capture in banner
  • 7. Side-by-side screenshot saved: Revit 3D view (left) + Panorama (right)
  • 8. Filename format: [ElementId]_[timestamp].png
  • 9. Saved to: %LOCALAPPDATA%\EliteTools\EliteScan\Screenshots\

Upload button workflow

  • 1. Click Upload in element row
  • 2. File dialog opens (PNG, JPG, BMP supported)
  • 3. User selects an external image (e.g., site photo, markup)
  • 4. File is copied to EliteScan screenshots folder
  • 5. Filename renamed: [ElementId]_[timestamp].png
  • 6. Element record updated with screenshot path
  • 7. Image badge turns green in element row
  • Use case: Link field photos, RFI images, or pre-existing documentation

Screenshots are embedded as base64 in HTML reports and displayed in the Verify Element Dialog preview. They are NOT stored in the JSON file itself, only the file path reference.

EliteScan Tutorial 22

360° Panorama Viewer — View Sync

Viewer > 360 Viewer

SYNC toggle enables bidirectional view synchronization between Revit's 3D viewport and the panorama viewer. Requires a perspective 3D view named "Pano View" (auto-created if missing).

How View Sync works

  • Creates "Pano View" — 3D perspective view with 60° FOV if it doesn't exist. Reused across sessions.
  • Sets camera position — Moves to selected scan's XYZ coordinates with 5.5 ft eye height offset.
  • Sets view direction — Matches panorama's yaw (horizontal rotation) and pitch (vertical angle).
  • Updates on mouse release — Listens for panorama drag events, syncs Revit camera after user stops dragging (avoids lag).
  • Right-click indicator — Draws temporary red line in Revit showing exact panorama view direction.
  • Bidirectional — Resync button pushes Revit camera direction back to panorama.

Sync only works in the "Pano View" 3D view. If user switches to a different view while SYNC is enabled, sync pauses and resumes when returning to Pano View.

Overlay & Quick Visualization

Visual feedback of analysis results directly in Revit's 3D viewport

Show Overlay & Clear Overlay

Overlay

Show Overlay applies color overrides to elements based on their analysis scores. Clear Overlay removes all color overrides.

Show Overlay

  • Loads most recent results from %LOCALAPPDATA%\EliteTools\EliteScan\Results\
  • Colors by score: A = green (#4CAF50), B = light green (#8BC34A), C = yellow (#FDD835), D = orange (#FF9800), F = red (#F44336), Reviewed = cyan (#5ED5DD)
  • Uses OverrideGraphicSettings to set element fill/projection colors in active view
  • Applies to all categories in the analysis results
  • Non-destructive — Original materials preserved, only view overrides applied
  • Persists across sessions — Overrides saved with view

Clear Overlay

  • Removes all color overrides from active view
  • Resets to default shading (materials and categories restore)
  • Does NOT delete analysis results (only removes visualization)
  • Quick toggle — Use to compare "before/after" overlays

Quick Deviations vs Show Deviations: Quick Deviations is a lightweight point cloud sample for fast checks. Show Deviations loads full per-point deviation data from .points.bin files for accurate color gradients (green=accurate, red=deviated).

Combine Analyses

Merge several saved per-area analyses into one consolidated whole-building report — new in v1.3.0

Combine Analyses — Selection Dialog

Results Window > Combine Analyses

Large projects are often analyzed area by area (one analysis per floor or zone). The Combine Analyses button in the results window (next to Export CSV) lets you pick two or more of those saved analyses and merge them into a single consolidated report that opens just like a whole-building run.

ControlWhat it does
Analysis checklist Checkbox list of saved analyses for this project. Each row shows the analysis date, its grading signature (tolerance and grade thresholds), and element/scan counts so you can tell them apart.
First selection locks settings The first analysis you check sets the grading signature for the combine. Analyses with a different tolerance or thresholds are then disabled (greyed out) so you can't mix incompatible grading.
Combine Enabled once 2 or more compatible analyses are checked. Builds the consolidated result and opens it in the results window.
Cancel Closes the dialog without combining anything.

EliteScan Tutorial 34b

Combine Analyses — How the merge resolves

Results Window > Combine Analyses

When the same scan location appears in more than one selected analysis, the merge keeps the newest capture. Scan locations that appear in only one analysis are simply carried through. The result reads exactly like a single whole-building analysis.

Merge rules

  • Newest wins per scan — if a scan location is in two analyses, the most recent one is kept; older captures are discarded
  • Point count is irrelevant — recency decides, not which run had more points
  • Different scans are unioned — scan locations unique to each analysis are all included
  • Grading signature gate — the first selection locks tolerance and thresholds; incompatible analyses are disabled in the dialog

What carries over

  • Manual approvals — QC approvals carry across the combine (they are project-level; see Section 05)
  • Opens like a normal run — discipline/category tree, scoring modes, export, and overlay all work as usual
  • Saved separately — written as EliteScan_consolidated_[timestamp].json; the source analyses are untouched

Requirements

  • At least 2 saved analyses exist for the project (Run Analysis on each area first)
  • Matching grading settings — same tolerance and grade thresholds; analyses that differ are disabled in the dialog

File Storage Locations

Where EliteScan stores settings, results, temporary data, and logs

File Storage Paths

All EliteScan data is stored under %LOCALAPPDATA%\EliteTools\EliteScan\ (typically C:\Users\[Username]\AppData\Local\EliteTools\EliteScan\).

PathContents
Settings\ User preferences JSON file. Stores tolerance, thresholds, CloudCompare path, included categories, scan family settings.
Results\ Analysis result JSON files. Each file is timestamped: [ProjectName]_[timestamp].json. Contains element scores, deviations, scan selections, verification data. Merged reports from Combine Analyses (Section 12) are saved here as EliteScan_consolidated_[timestamp].json.
[Project]\Scan\calibration\ New in v1.3.0. Per-scan E57 calibration cache, one file per point cloud: [rcpKey].json. Created when you click Calibrate in Settings (Section 03). Validated before every analysis; re-Calibrate if the point cloud moves. Project-scoped path: %LOCALAPPDATA%\EliteTools\[Project]\Scan\calibration\.
[Project]\Scan\review-store.json New in v1.3.0. Durable per-project QC review store. Holds your element approvals so they survive re-analysis and model fixes; conflicts are flagged, never lost (Section 05). Project-scoped path: %LOCALAPPDATA%\EliteTools\[Project]\Scan\review-store.json.
TempGeometry\ E57 geometry exports. Created during Run Analysis (Step 1). Contains mesh representations of Revit elements. Auto-cleaned after analysis.
TempClouds\ E57 point cloud crops. Created during Run Analysis (Step 2). Contains radius-cropped point clouds. Auto-cleaned after analysis.
DeviationData\ .points.bin files with per-point deviation scalars. Created during Run Analysis (Step 4). Used by Show Deviations visualization. Persistent.
Screenshots\ Verification screenshots. Filename format: [ElementId]_[timestamp].png. Side-by-side Revit + Panorama captures or uploaded images.
Logs\ Diagnostic log files. One per day: EliteScan_[date].log. Contains CloudCompare output, errors, performance metrics.
Templates\ Saved settings templates. JSON files with user-defined presets (e.g., "Mechanical Only", "Tight Tolerance").

To reset EliteScan to factory defaults, delete the entire %LOCALAPPDATA%\EliteTools\EliteScan\ folder. Settings and templates will regenerate on next launch.

Troubleshooting

Common issues and solutions

Troubleshooting Guide

CloudCompare Not Found

  • Symptom: Run Analysis fails with "CloudCompare.exe not found" error
  • Solution: Open Settings, verify CloudCompare Path points to valid exe
  • Download: cloudcompare.org (free, open-source)
  • Default path: C:\Program Files\CloudCompare\CloudCompare.exe

No Scans Found

  • Symptom: Place Scans reports "No scans found in point cloud"
  • Causes: RCP has no linked RCS files, RCS files missing from disk, wrong Point Cloud Root path
  • Solution: Verify RCS files exist, check RCP properties, set Point Cloud Root in Settings

Analysis Stops: Calibration Missing or Stale

  • Symptom: Run Analysis stops with a message naming a point cloud (not set up, can't read scans, or stale)
  • Cause: v1.3.0 needs each point cloud associated with its per-scan E57 folder and calibrated; or the cloud was moved since calibrating
  • Solution: Run E57ImageExtractor, then in Settings associate the _scans folder and click Calibrate for that point cloud (re-Calibrate if moved). See Section 03
  • Note: There is no fallback to the old merged-cloud method by design

Panorama Images Not Loading

  • Symptom: 360 Viewer shows "No panorama found" message
  • FARO mode: Verify original FARO scan folders with panorama_*.jpg files are accessible
  • E57 mode: Verify E57 panorama images are available for the scans
  • Wrong orientation: Orientation is baked at Place time in v1.3.0 — re-run Place Scans with the correct mode (Section 02), not a Settings toggle

View Sync Not Working

  • Symptom: SYNC toggle enabled but panorama doesn't follow Revit camera
  • Cause: Active view is not "Pano View" 3D perspective view
  • Solution: Click Resync to auto-create "Pano View" or manually switch to it

Analysis Results Not Loading

  • Symptom: View Report opens empty window or shows "No results found"
  • Cause: No completed analyses, corrupted JSON, results folder moved
  • Solution: Check %LOCALAPPDATA%\EliteTools\EliteScan\Results\ for .json files, re-run analysis if needed

Out of Memory During Analysis

  • Symptom: Revit crashes or analysis fails on large projects
  • Cause: Too many elements, large point clouds, insufficient RAM
  • Solution: Reduce AnalysisRadius, analyze fewer scans at once, exclude non-critical categories in Settings
