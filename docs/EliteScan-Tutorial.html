Section 01

Ribbon overview

The "Elite Scan" tab with four panels: Analysis, Viewer, Overlay, and Point Clouds

Ribbon panels & buttons

Elite Scan

EliteScan installs a dedicated "Elite Scan" tab in the Revit ribbon. It contains four panels grouping point cloud analysis, visualization, and data management tools. Supports Revit 2022 through 2026.

Analysis panel

  • Place Scans — Places scan-location markers from the point cloud's scan origins
  • Analyze View — Element-based deviation analysis of the active view
  • View Report — Opens the results window
  • Settings — Configure tolerance, scoring thresholds, and categories

Viewer panel

  • 360 Viewer — Dockable equirectangular panorama pane

Overlay panel

  • Show Overlay — Colors elements by score (A-F grades)
  • Clear Overlay — Removes all color overrides

Point Clouds panel

  • Cloud Panel — Dockable list of every point cloud in the project; add, convert, show/hide, and remove clouds here
  • Revit 2 ReCap — Shows point cloud transform data with per-value copy buttons
  • Render Settings — Live point density, size, and color controls
  • Cloud Reset — Resets a cloud's placement back to origin
  • Isolate to Element — Clips the cloud to the selected element(s)
  • Clear Isolation — Shows the full cloud again

Icons auto-swap between Light and Dark variants to match Revit's current UI theme. The add-in uses ricaun.Revit.UI for ribbon creation with automatic DPI scaling.

New in v2.0.0: deviation analysis is now element-based — Analyze View scores every element in the active view (or your selection) directly against the point cloud using a built-in multi-core engine, with no CloudCompare and no per-scan E57 setup. Point clouds are added and managed from the new Cloud Panel, displayed by a fast built-in renderer with live Render Settings, and can be clipped to a single element with Isolate to Element.

New in v2.0.1: adding a cloud converts it in the background inside Revit — a preview appears immediately and swaps to the full renderer when conversion finishes, with a live progress bar and an upfront time estimate. The separate Load Point Cloud button is retired; everything lives in the Cloud Panel.

Section 02

Analysis panel — Place Scans

Auto-places Scan Location family instances at each scan origin, with origin and orientation read straight from the ReCap point cloud

Place Scans button

Analysis > Place Scans

Before placing scans, add the point cloud for the project in the Cloud Panel (see Section 03 — Point clouds). Automatically places Scan Location family instances at each scan origin. Both the origin position and the scanner's viewing orientation are read straight from the ReCap point cloud — there is no FARO/E57 mode prompt to choose. Each instance is populated with the metadata used by analysis and the 360 Viewer.

Requirements

  • Point cloud loaded for the project (see Section 03)
  • Scan Location family loaded in the project (auto-prompts if missing)
  • Levels defined in the model

What it sets

  • ScanID — Scan name from the point cloud
  • Level — Closest Revit level by elevation
  • SupportFolderPath — Where the scan's panorama image lives
  • Location — XYZ coordinates of the scan origin
  • Viewing orientation — Read from the ReCap scan pose and baked onto the instance (used by the 360 Viewer)

Re-runnable: instances are matched by their ScanID parameter. Re-running Place Scans updates existing markers in place (re-baking position and orientation from the cloud) and places new ones for any scans that don't have a marker yet — it no longer skips them.

Panorama extraction choice at place time

  • Extract all now — loop every scan and write all panorama images up front (shows a progress bar; you can cancel)
  • Extract lazily — skip extraction now; each scan's panorama is pulled from the cloud the first time you open it in the 360 Viewer
  • Either way — only structured scans have a panorama; the viewing orientation is baked here regardless of this choice (see Section 08)

Section 03

Point clouds — load, manage & display

Add an .rcp and convert it in the background, manage every cloud in the project, and tune how the built-in renderer draws them — all on the Point Clouds panel

Add a cloud — convert in the background

Point Clouds > Cloud Panel > Add

Pick an .rcp file in the Cloud Panel to bring a scan into the project. EliteScan validates the file through the ReCap SDK and checks the cloud's render cache — the .potree folder the built-in renderer draws from. If the cache is missing, EliteScan offers to convert it in the background right inside Revit, with an upfront time estimate.

What happens when you Add

  • Preview appears immediately — the cloud shows via Revit's native display while the cache builds
  • Background conversion with a live progress bar on the cloud's row
  • Automatic swap — when conversion finishes, the high-performance renderer takes over, no reload needed

Your choices at Add time

  • Convert in the background — the recommended path described here
  • Use the standalone converter — keep the preview and build the cache outside Revit (e.g. a scanning-team machine)
  • Cancel — the cloud stays listed as a preview row; convert it later with Build cache

A cache that is already built loads instantly — no prompt. Once the render cache is ready, both display and Analyze View (Section 04) work against it.

Cloud Panel — manage every cloud in the project

Point Clouds > Cloud Panel

A dockable panel that lists every point cloud in the project — including preview rows (canceled conversions) and clouds inserted with Revit's own Insert > Point Cloud. Each row shows the cloud's name, its file path, and its status: linked, source missing, no cache, or converting (with a live progress bar).

ButtonWhat it does
Add Brings another point cloud into the project and offers background conversion if its render cache is missing (previous slide).
Build cache Converts a row that has no render cache yet (e.g. a canceled convert or an Insert > Point Cloud link). Every build asks for confirmation first with a size and time estimate — fresh builds default to Yes, rebuilding an existing cache defaults to No.
Cancel Stops tracking an in-progress conversion; the row stays listed as a preview so you can Build cache later.

Re-locate Point a row at the cloud's new file location after the source .rcp has moved — clears the "source missing" status without losing any results.
Remove Drops the cloud from the project list. Does not delete the source file on disk.

The panel is the single place to see, at a glance, which clouds are linked, converting, or need attention before you analyze or view them.

EliteScan Tutorial 5c

Render Settings — live display dials

Point Clouds > Render Settings

A modeless panel with live dials over the built-in point-cloud renderer. Every change applies instantly — no reload and no re-conversion, just a redraw. These settings control how the cloud looks, not how it's analyzed.

ControlWhat it does
Density How many points the renderer draws. Presets Low (1M), Medium (3M, default), High (6M), Ultra (10M), plus a fine slider. Higher density shows more detail where the camera needs it.
Point size How big each point draws on screen, 1–5 px. Larger points fill gaps in sparse areas; smaller points show fine structure.
Style — Normal Default. Draws the cloud solid and opaque.
Style — X-ray Draws the cloud semi-transparent so you can read model geometry and structure through it. The Contrast slider controls how see-through it gets. Works with any color mode.
Color — RGB Default. Draws each point in its captured scan color.
Color — Elevation Colors points by height (Z) through a blue-to-red ramp. Use the Use cloud bounds toggle to auto-range to the cloud, or type a manual Min / Max Z in feet to color a specific band.
Color — Uniform Paints the whole cloud one color you pick by hex value (#RRGGBB), with a preview swatch. Handy for clean overlays and presentation views.
Color — Intensity Colors points by the scanner's raw return brightness (grayscale), which makes surfaces, materials, and markings much easier to read. Needs a cache built with v2.1.0 or newer — see the note below.
Color — Normal shading Lights the cloud like a headlight so shape and depth stand out clearly, even on single-color scans. Needs a cache built with v2.1.0 or newer — see the note below.
Deviation heatmap After running Analyze View (Section 04), colors every point by how far it sits from the model — green within tolerance, shading to red at the outlier limit, dim gray where nothing was measured. Layers on top of any style, including X-ray. Off by default.

Intensity & Normal shading need a v2.1.0 cache

  • These two color modes read extra per-point data (scanner intensity and surface normals) that the v2.1.0 converter records into the render cache
  • A cloud built by an older converter won't have it — the radio button stays disabled with a tooltip telling you to rebuild
  • Rebuild from the Cloud Panel (Section 03) — Build cache reconverts the cloud and adds the new data

Display settings live here, separate from the analysis Settings dialog (Section 10), so changing how a cloud looks never touches your tolerance or category setup.

Section 04

Analysis panel — Analyze View

One-click deviation analysis of the active view against the point cloud, using the built-in multi-core cloud-to-mesh engine

Analyze View — what it does

Analysis > Analyze View

One click scores every model element in the active view — or, if you have elements selected, just the current selection — against the loaded point cloud(s). There's no scan-picking dialog, no CloudCompare, no E57 files, and no calibration step. Each element's deviation is simply the distance from the cloud points in its region to that element's own mesh, with a 2-inch outlier cut to keep stray points from skewing the score.

How it works

  • Active view or selection — scores every in-scope model element you can see (or just what you've selected)
  • Built-in engine — a multi-core, multi-process cloud-to-mesh comparison runs entirely inside EliteScan
  • Per-element distance — each element is measured against its own mesh using the cloud points in its region
  • 2-inch outlier cut — points far outside the element's region are dropped before scoring

Multi-cloud, multi-scan results

  • Each loaded cloud is a "scan" source — the existing multi-scan results UX still applies
  • Best-visibility auto-pick — an element shown by several clouds defaults to the one that saw it best
  • Per-element scan dropdown — switch which cloud's reading an element is graded on (see Section 05)
  • Grades from % in tolerance — the share of an element's points within tolerance, against your A/B/C/D thresholds

Results auto-save and open in the Results Window (Section 05), exactly as before — only the engine underneath changed. After a run, switch on the Deviation heatmap in Render Settings (Section 03) to color the cloud itself by distance to the model.

EliteScan Tutorial 7

Analyze View — before you run it

Analysis > Analyze View

Analyze View needs a converted point cloud and a model view. If neither is ready it stops with a clear message before doing any work. A modal progress dialog shows live counts (elements and points scored) and a cancel button while it runs across all CPU cores.

Required before it runs

  • At least one converted cloud — a point cloud whose .potree render cache is built by the EliteScan Converter. If none is converted, it aborts with a "Convert Point Cloud First" message
  • A model view active — 3D, plan, section, or elevation. It won't run on a schedule, sheet, or view template
  • Included categories & tolerance set in Settings (Section 10) — only in-scope categories are scored

While it runs

  • Tessellates each element in scope into its own mesh
  • Reads the cloud points in each element's region and measures distance to that mesh
  • Scores & grades each element from % of points within tolerance, then aggregates to category and discipline
  • Cancel any time — the run stops cleanly; results auto-save when it completes

Because every loaded cloud runs as its own batch, you get one result set per cloud and the multi-scan results UX picks the best reading per element automatically.

Results Window — What's new in v1.3.0

Analysis > View Report

Several results-window changes land in v1.3.0: a live point-count filter, durable per-project review state, and a correction to how reviewed elements affect category accuracy.

ChangeWhat it means for you
Minimum-point filter (live) A toggle in the results window hides elements/scans at or below a point-count threshold (set in Settings). By default it only hides zero-point "No Data" rows so the list shows what was actually scanned. Toggling it recalculates the visible list immediately.

  • Conflicts surfaced — differing approvals for the same element become a conflict that counts as unreviewed until a person resolves it; nothing is silently discarded
  • First open migrates history — existing approvals from past results files are seeded into the project review store automatically

Section 06

Scoring System

Letter grade thresholds and multi-scan scoring modes

Scoring — Grade Thresholds

Settings

Elements receive letter grades (A-F) based on the percentage of captured points within the tolerance threshold (default 1 inch). Verified elements receive a "Reviewed" grade.

A — 90%+ in tolerance
B — 80%+ in tolerance
C — 70%+ in tolerance
D — 60%+ in tolerance
F — Below 60%
Reviewed — Verified by user

Thresholds are fully configurable in Settings. Tolerance defines the acceptable deviation distance (default 1 inch). Grade thresholds define the percentage cutoffs for A/B/C/D/F.

Multi-scan scoring modes

  • Best Visibility — Uses scan with most captured points. Default. Prioritizes data completeness.
  • Best Score — Uses scan with highest grade. Optimistic. Good for "prove it's right" scenarios.
  • Top 10 Average — Averages scores from top 10 scans by point count. Balanced. Reduces outlier influence.
  • Custom — Allows per-element scan selection. Maximum control. Use for investigating conflicts.

EliteScan Tutorial 18

Verification Screenshot Workflow

Results Window

Two methods for associating screenshots with verified elements: automated Capture workflow or manual Upload workflow.

Capture button workflow

  • 1. Click Capture in element row
  • 2. Panorama Viewer opens if not already visible
  • 3. Banner appears at top: "Navigate to view element. Press Capture when ready."
  • 4. Auto-zooms Revit viewport to element
  • 5. User navigates the 360° panorama to show element
  • 6. User clicks Capture in banner
  • 7. Side-by-side screenshot saved: Revit 3D view (left) + Panorama (right)
  • 8. Filename format: [ElementId]_[timestamp].png
  • 9. Saved to: %LOCALAPPDATA%\EliteTools\{Project}\Scan\Screenshots\

Upload button workflow

  • 1. Click Upload in element row
  • 2. File dialog opens (PNG, JPG, BMP supported)
  • 3. User selects an external image (e.g., site photo, markup)
  • 4. File is copied to EliteScan screenshots folder
  • 5. Filename renamed: [ElementId]_[timestamp].png
  • 6. Element record updated with screenshot path
  • 7. Image badge turns green in element row
  • Use case: Link field photos, RFI images, or pre-existing documentation

Screenshots are embedded as base64 in HTML reports and displayed in the Verify Element Dialog preview. They are NOT stored in the JSON file itself, only the file path reference.

EliteScan Tutorial 22

360° Panorama Viewer — View Sync

Viewer > 360 Viewer

SYNC toggle enables bidirectional view synchronization between Revit's 3D viewport and the panorama viewer. Requires a perspective 3D view named "Pano View" (auto-created if missing).

How View Sync works

  • Creates "Pano View" — 3D perspective view if it doesn't exist. Reused across sessions.
  • Sets camera position — Moves to the selected scan's exact XYZ position (no eye-height offset).
  • Sets view direction — Matches panorama's yaw (horizontal rotation) and pitch (vertical angle).
  • Updates on mouse release — Listens for panorama drag events, syncs Revit camera after user stops dragging (avoids lag).
  • Right-click indicator — Draws temporary red line in Revit showing exact panorama view direction.
  • Bidirectional — Resync button pushes Revit camera direction back to panorama.

Sync only works in the "Pano View" 3D view. If user switches to a different view while SYNC is enabled, sync pauses and resumes when returning to Pano View.

Overlay & Quick Visualization

Visual feedback of analysis results directly in Revit's 3D viewport

Show Overlay & Clear Overlay

Overlay

Show Overlay applies color overrides to elements based on their analysis scores. Clear Overlay removes all color overrides.

Show Overlay

  • Loads most recent results from %LOCALAPPDATA%\EliteTools\{Project}\Scan\Results\ (the results Analyze View produces)
  • Colors by score: A = green (#4CAF50), B = light green (#8BC34A), C = yellow (#FDD835), D = orange (#FF9800), F = red (#F44336), Reviewed = cyan (#5ED5DD)
  • Uses OverrideGraphicSettings to set element fill/projection colors in active view
  • Applies to all categories in the analysis results
  • Non-destructive — Original materials preserved, only view overrides applied
  • Persists across sessions — Overrides saved with view

Clear Overlay

  • Removes all color overrides from active view
  • Resets to default shading (materials and categories restore)
  • Does NOT delete analysis results (only removes visualization)
  • Quick toggle — Use to compare "before/after" overlays

Show Overlay and Quick Deviations both read the saved per-element grades from the latest analysis — they color the model elements, not individual points (green = accurate, red = deviated; cyan = verified). To inspect the points around a specific element, use Isolate to Element on the Point Clouds panel (Section 03).

Outlier window (in) Numeric input (default 2 in). Points farther than this from an element's region are dropped before scoring, so stray points don't skew the grade. Used by Analyze View (Section 04).
Point Cloud Root Folder path input with Browse button. Root directory where point cloud files are stored. Used as a fallback when resolving a cloud's source location.
Minimum points to display Numeric input. Elements and scans at or below this point count are hidden from results. Default is 0, which only hides zero-point "No Data" rows. Raise it to drop sparsely-captured elements from the report. A live toggle in the results window honors this value.

No external tools: Analyze View uses EliteScan's built-in cloud-to-mesh engine — there is no CloudCompare path to configure and no per-scan E57 setup. It only needs a converted point cloud (Section 03–04).

Point-cloud display (density, point size, color) is not configured here — it lives in the separate Render Settings panel on the Point Clouds panel (Section 03), so changing how a cloud looks never touches your analysis setup.

Large projects are often analyzed area by area (one analysis per floor or zone). The Combine Analyses button in the results window (next to Export CSV) lets you pick two or more of those saved analyses and merge them into a single consolidated report that opens just like a whole-building run.

ControlWhat it does

EliteScan Tutorial 34b

Combine Analyses — How the merge resolves

Results Window > Combine Analyses

The merge resolves element by element: every element from every selected analysis is kept, and when the same element appears in more than one, the newest capture wins. Elements that have since been deleted from the model are skipped (the completion message says how many). The result reads exactly like a single whole-building analysis.

Merge rules

  • Newest wins per element — if an element was captured in two analyses, the most recent capture is kept; an element in only one analysis is carried through
  • Point count is irrelevant — recency decides, not which run had more points
  • Deleted elements are dropped — elements no longer in the model are removed from the merged report and counted in the completion message
  • Grading signature gate — the first selection locks tolerance and thresholds; incompatible analyses are disabled in the dialog

What carries over

  • Manual approvals — QC approvals carry across the combine (they are project-level; see Section 05)
  • Opens like a normal run — discipline/category tree, scoring modes, export, and overlay all work as usual
  • Saved separately — written as EliteScan_consolidated_[timestamp].json; the source analyses are untouched

Requirements

  • At least 2 saved analyses exist for the project (run Analyze View on each area first)
  • Matching grading settings — same tolerance and grade thresholds; analyses that differ are disabled in the dialog

File Storage Locations

Where EliteScan stores settings, results, temporary data, and logs

File Storage Paths

EliteScan data is stored under %LOCALAPPDATA%\EliteTools\, split into global data shared across projects (Global\Scan\) and per-project data ({Project}\Scan\, named from the Revit document title).

PathContents
Global\Scan\settings.json Global analysis settings. Stores tolerance, A/B/C/D thresholds, outlier window, included categories, exclusion filters, point-cloud root, and scan-family settings.
Global\Scan\render-settings.json Render Settings panel state (Section 03): point density, point size, and color mode. Kept separate from settings.json so display changes never touch analysis setup.
Global\Scan\Templates\ Saved settings templates. JSON files with user-defined presets (e.g., "Mechanical Only", "Tight Tolerance"). Shared across projects.
{Project}\Scan\Results\ Analysis result JSON files, timestamped: EliteScan_[tol]in_[timestamp].json. Contains element scores, deviations, scan selections, and verification data. Merged reports from Combine Analyses (Section 12) are saved here as EliteScan_consolidated_[timestamp].json.
{Project}\Scan\review-store.json Durable per-project QC review store. Holds your element approvals so they survive re-analysis and model fixes; conflicts are flagged, never lost (Section 05).
{Project}\Scan\Screenshots\ Verification screenshots. Filename format: [ElementId]_[timestamp].png. Side-by-side Revit + Panorama captures or uploaded images.
Global\Scan\Logs\ & {Project}\Scan\Logs\ Diagnostic log files, one per day: EliteScan_[date].log. Contains analysis steps, errors, and performance metrics.
<rcpName> Support\ (beside the cloud) Panorama JPGs, one per structured scan: [scanId].jpg. These live next to the point cloud on disk — not under EliteTools — written by Place Scans or lazily by the 360 Viewer (Section 08).

To reset global EliteScan settings to defaults, delete %LOCALAPPDATA%\EliteTools\Global\Scan\; settings and templates regenerate on next launch. Per-project results and approvals live under %LOCALAPPDATA%\EliteTools\{Project}\Scan\.

Troubleshooting

Common issues and solutions

Troubleshooting Guide

Analyze View: "Convert Point Cloud First"

  • Symptom: Analyze View stops immediately and tells you to convert the cloud
  • Cause: No point cloud has a built render cache (the .potree folder)
  • Solution: Click Build cache on the cloud's row in the Cloud Panel (or use the standalone EliteScan Converter), then re-run Analyze View. See Section 03–04

Point Cloud Won't Display

  • Symptom: A loaded cloud doesn't draw in the 3D or 2D view
  • Causes: No render cache yet, conversion still in progress, or an unsupported view (schedule/sheet/template)
  • Solution: Check the cloud's state in the Cloud Panel; click Build cache if needed; make sure the cloud isn't hidden (Show / Hide buttons); switch to a 3D, plan, section, or elevation view (Section 03)

No Scans Found

  • Symptom: Place Scans reports "No scans found in point cloud"
  • Causes: The point cloud isn't loaded, its .rcp source has moved, or it holds no scan origins
  • Solution: Load the cloud (or Re-locate it in the Cloud Panel), confirm the .rcp path is valid, then re-run Place Scans (Section 02–03)

Panorama Images Not Loading

  • Symptom: 360 Viewer shows "No panorama available" message
  • Cause: The cloud isn't loaded, Place Scans hasn't run, or the scan is not structured (only structured scans have a panorama)
  • Solution: Add the point cloud in the Cloud Panel and run Place Scans (Section 02); panoramas are pulled from the cloud, lazily on first open. Orientation is baked at Place time — re-run Place Scans if it looks off (Section 08)

View Sync Not Working

  • Symptom: SYNC toggle enabled but panorama doesn't follow Revit camera
  • Cause: Active view is not "Pano View" 3D perspective view
  • Solution: Click Resync to auto-create "Pano View" or manually switch to it

Analysis Results Not Loading

  • Symptom: View Report opens empty window or shows "No results found"
  • Cause: No completed analyses, corrupted JSON, results folder moved
  • Solution: Check %LOCALAPPDATA%\EliteTools\{Project}\Scan\Results\ for .json files, re-run Analyze View if needed

Out of Memory During Analysis

  • Symptom: Revit crashes or analysis fails on large projects
  • Cause: Too many elements in scope, very large point clouds, insufficient RAM
  • Solution: Analyze a smaller view or selection at a time, exclude non-critical categories in Settings, and isolate to the area you're checking
