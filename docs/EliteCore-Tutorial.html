Section 01

Ribbon overview

The “Elite Core” tab with four panels: Project Setup, Auto Sync, Tab Colors, and Export

Ribbon panels & buttons

Elite Core

EliteCore installs a dedicated “Elite Core” tab in the Revit ribbon. It contains four panels grouping core utilities for project setup, automatic syncing, tab color customization, and batch export. Supports Revit 2022 through 2026.

Project Setup panel

  • User Workset — Create and activate personal workset
  • Link Worksets — Organize all links onto dedicated worksets
  • Setup Wizard — Shared parameter binding manager

Auto Sync panel

  • Auto Sync Settings — Configure sync intervals and behavior
  • Sync Log — View sync history and statistics

Tab Colors panel

  • Tab Color Settings — Configure document tab color scheme
  • Toggle Colors — Enable or disable tab colors instantly

Export panel

  • Batch Export — Export sheets and views to PDF, DWG, and NWC

Icons auto-swap between Light and Dark variants to match Revit’s current UI theme. The add-in uses ricaun.Revit.UI for ribbon creation with automatic DPI scaling.

Section 02

Project Setup — User Workset

Create and activate a personal workset named after your Autodesk username

User Workset command

Project Setup > User Workset
Action Details
Creates workset Named 01 - {username} using your Revit username
Sets active Immediately switches to the new workset as the active workset
Opens dialog Launches the native Revit Worksets dialog for verification
Skips duplicates If the workset already exists, simply activates it without creating a new one

Auto-activation on document open

  • Automatic: When any workshared document opens, EliteCore searches for a 01 - {username} workset
  • Silent: If found, it is set as active automatically — no dialog or notification
  • Always on: This happens every time, ensuring you always start on your personal workset

Requirements

  • Worksharing must be enabled — if not, shows message: “This command requires a workshared document. Enable worksharing first.”

Section 06

Auto-Sync — How it works

Background service that monitors your document and triggers syncs automatically

Sync status states

Auto Sync

The Auto-Sync service runs in the background and tracks the sync status of every open workshared document. Status changes based on time since last sync and the thresholds configured in settings.

Synced
Warning
Overdue
Syncing
Failed
N/A
Event Service response
Idling Checked every 30 seconds — evaluates if interval sync is due or idle threshold exceeded
Document Synced Updates last sync time, logs sync to history, clears any failed status
Document Closing If pre-close reminder enabled and sync overdue: shows dialog “It’s been X minutes since your last sync. Close anyway?”
Document Saved If sync-on-save enabled, triggers synchronization immediately
Document Changed Updates last activity timestamp for idle tracking
Document Opened Initializes tracking, loads settings, seeds last sync time to prevent immediate sync

Sync execution flow

Auto Sync

When a sync is triggered (by interval, idle, or save), the service follows this sequence:

Execution steps

  • 1. Guard check — Verify document isn’t already syncing (prevents re-entry)
  • 2. Status update — Set status to Syncing
  • 3. Build options — Configure relinquish settings, compact flag, and comment (“Auto-sync (interval)”)
  • 4. Synchronize — Call doc.SynchronizeWithCentral()
  • 5. Measure — Record duration with Stopwatch
  • 6. Log result — Write success/failure to sync log with duration and type
  • 7. Notify — If not in quiet mode, show status notification on failure

Order palette & filter rules

Tab Colors > Settings

Order-based palette

  • Color list — 10 default colors shown as swatches with hex codes
  • Add — Append a new color to the palette
  • Remove — Delete the selected color
  • Up / Down — Reorder colors in the palette
  • Wraps around — When palette is exhausted, cycles back to first color

Filter-based rules

  • Color swatch — Click to change the rule’s color

Section 09

Tab Colors — How it works

Behind the scenes: MFC pointer matching, color assignment, and visual style rendering

Technical overview

Tab Colors

Tab coloring works by matching Revit’s internal MFC document pointers to WPF tab controls, then applying custom styles. Uses the same approach as pyRevit for reliable tab identification.

MFC document pointer matching

  • API → MFC: Uses reflection to get doc.getMFCDoc().GetPointerValue() → unique IntPtr per document
  • Tab → MFC: Navigates TabItem → DataContext → LayoutDocument → Content → MFCMDIChildFrameControl → IntPtr
  • Match: Builds a map of MFC ID → Document Title from the API, then matches each TabItem by its MFC pointer

Order-based assignment

  • Sequential — First project gets palette[0], second gets palette[1], etc.
  • Separate cycles — Family docs cycle palette independently from projects
  • Wraps around — When palette exhausted, returns to start

Filter-based assignment

  • Top-to-bottom — Rules evaluated in UI order
  • First match wins — Stop at first matching regex
  • Case-insensitive — Pattern matching ignores case
  • Fallback — Use .* as last rule to catch all unmatched docs

Visual style rendering

Tab Colors
Style Selected tab Unselected tab
TopBar 3px colored bar on top edge, full opacity 3px colored bar, 50% opacity
Background Full color fill, auto-contrast text (white on dark, black on light) Opacity-controlled fill (default 85%), hover lightens by 30 RGB
Border 2px colored left/top/right border, full opacity 1px border, 50% opacity

Toggle Colors button

  • Instant toggle — No dialog; immediately enables or disables tab coloring
  • Off: Removes all tab colors, unsubscribes from Idling event
  • On: Subscribes to Idling event, colors apply within 500ms
  • Preserves settings — Only changes enabled state, all other settings remain

The service hooks into Revit’s Idling event (throttled to 500ms intervals) to discover tabs, match documents, and apply/update styles.

Section 10

Export — Batch Export

Export multiple sheets and views to PDF, DWG, and NWC in a single operation with reusable profiles

Batch Export wizard

Export > Batch Export

A 3-step wizard for exporting sheets and views in bulk. Select what to export, configure format-specific settings, and choose your output folder. All settings are saved to reusable profiles.

Step Name What you do
1 Select Choose sheets and/or views to export. Use search, view type filter, and View/Sheet Sets to manage selections. Set up custom file naming.
2 Format Pick one or more formats (PDF, DWG, NWC) using checkbox-tabs. Configure settings for each format individually.
3 Output Choose an output folder, review the export summary, and start the export.

Requirements

  • Active Revit project — must have a project open (not a family document)
  • Sheets or views — at least one sheet or view must exist in the project
  • DWG Export Setup — a named DWG export setup must be configured in Revit (for DWG format only)

Step 1: Select sheets & views

Batch Export > Select

Toggle between Sheets and Views mode using the radio buttons. Check individual items or use Select All. Both modes maintain their own selections — you can export sheets and views together in a single operation.

Control Details
Source toggle Sheets Views — switch between sheet list and view list; selections are preserved in both
Search Type to filter the list by name, number, or any visible column text
View Type filter In Views mode, a dropdown in the Type column header filters by view type (3D View, Floor Plan, Detail, Elevation, etc.)
File Name button Click “File Name” in the column header to open the naming editor popup
Select Set Save and load named selection sets. Create, update, or delete sets from the dropdown.
Status bar Shows selected/total counts for both modes: Sheets: 3/20 • Views: 2/15

File naming editor

Batch Export > Select

Build custom file names from tokens. Tokens are displayed as chips and can be rearranged or removed. Separate naming patterns are saved for sheets and views.

Token Type Description Examples
Parameter A Revit sheet/view parameter value Sheet Number, Sheet Name, View Name, Current Revision
Separator A fixed character between tokens - _ - .
Variable A dynamic value resolved at export time Date, Project Name, Project Number
Literal Custom text you type in directly Any prefix, suffix, or label text

Example result: 02_J25-188_M-001_REV0 from tokens: 02_ Project Number _ Sheet Number _ REV Current Revision

Step 2: Format settings

Batch Export > Format

Check one or more format tabs to include them in the export. Click a tab to view and configure its settings. Each format has its own options panel.

PDF options

  • Color Depth — Color, Grayscale, or B&W
  • Raster Quality — Low, Medium, High, or Max
  • Paper Placement — Center or Lower Left
  • Zoom — Fit to Page or custom percentage
  • Processing — Vector or Raster
  • Appearance — Hide ref planes, scope boxes, crop boundaries, unreferenced tags
  • Combined file — Export all sheets into one PDF
  • View links in blue — Hyperlinks appear in blue

DWG options

  • Export Setup — Select a named DWG export setup configured in Revit
  • External Refs — Export views as separate xref files
  • Clean .pcp files — Delete plot config files after export

NWC options

  • Element Properties — Include element data
  • Export Links — Include linked models
  • Divide into Levels — Split by level
  • Export Parts — Include parts
  • Rooms — Export as attribute and/or geometry
  • Coordinates — Internal or Shared
  • Faceting — Geometry detail slider
  • Parameters — None, Elements, or All

Profiles & sharing

Batch Export

All export settings are saved to named profiles. Profiles are stored as individual JSON files that can be copied between machines or shared with team members.

Profile controls

  • Profile dropdown — Switch between saved profiles
  • Save — Save current settings to the active profile
  • + (New) — Opens the New Profile popup
  • Delete — Remove the active profile (cannot delete last profile)

New Profile popup

  • Profile Name — Enter a name for the new profile
  • Create Blank — New profile cloned from current settings
  • Load from File — Import settings from a JSON file (e.g., a profile shared by a teammate)

Profiles stored at: %LOCALAPPDATA%\EliteTools\Global\Core\BatchExportProfiles\

Section 13

Getting started

Recommended setup workflow for a new project

Step-by-step setup

Workflow
Step Action Details
1 Enable worksharing Go to Collaborate tab → Collaborate button → “Collaborate within your network”. Required for Project Setup and Auto-Sync features.
2 User Workset Click User Workset to create and activate your personal workset. This auto-activates on every future document open.
3 Link Worksets Click Link Worksets to organize all linked models, CAD files, point clouds, and coordination models onto dedicated worksets.
4 Setup Wizard Click Setup Wizard to review and bind shared parameters required by other Elite add-ins. Click “Add Missing Parameters” to create them all.
5 Configure Auto-Sync Click Auto Sync Settings to set your sync interval (30 min recommended), enable pre-close reminder, and choose relinquish options.
6 Customize Tab Colors Click Tab Color Settings to choose a visual style. TopBar is recommended for subtlety, Dimmed family mode for clarity.
7 Set up Batch Export Click Batch Export to create your first export profile. Configure file naming, format settings, and output folder. Save as a profile for future use.

Tips & advanced workflows

Workflow

Auto-Sync profiles

  • Large teams: 15-minute intervals to keep data fresh
  • Frequent changes: Enable sync-on-save for real-time updates
  • Strict standards: Enable compact central on sync
  • External coordination: Disable relinquish to hold element ownership

Filter rule examples

  • Project codes: ^230.* → Blue for all “230-XXX” projects
  • Disciplines: ^AR-.* → Blue, ^MEP-.* → Green
  • Test files: .*TEST.* → Red for easy identification
  • Catch-all: .* at bottom → default color for unmatched

Troubleshooting

  • Tab colors not updating: Click Toggle Colors twice to reset the service
  • Sync failing: Disable quiet mode to see error messages in the sync dialog
  • Workset not activating: Verify worksharing is enabled and you are authorized
  • Parameters not appearing: Check Setup Wizard — green dots mean bound, red dots mean missing
