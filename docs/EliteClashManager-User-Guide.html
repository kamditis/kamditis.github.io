Complete user guide -- every ribbon button, tab, dialog, and workflow for clash triage, spatial mapping, and ACC/BIM 360 issue publishing

User Guide • v1.16.1 • July 2026

What EliteClashManager does

EliteClashManager extends Navisworks Clash Detective with the capabilities coordination teams actually run their meetings on:

Triage & organize

  • Split / Distribute / Group clash groups by distance, level, or elevation
  • Bulk rename with find/replace and room-name placeholders
  • Status management with preview-then-push commits
  • Sort alphabetically, by spatial proximity, or by clash severity
  • Filter by status, viewpoint, issue, column text, or 3D selection

Publish & sync

  • One-click issue publishing to ACC / BIM 360 with screenshots, pushpins, and deep links
  • Status sync both directions between Clash Detective and cloud issues
  • Auto Issue pipeline -- a nine-phase automated triage run
  • Trade & level mappings that auto-fill issue type, subtype, assignee, and location

Spatial intelligence

  • Room Atlas -- maps every clash to its Revit room automatically
  • Room overlay & 3D markers rendered live in the viewport
  • Viewpoint generation with configurable section boxes and baked redlines
  • Persistent data -- issue numbers and room names survive file save/reopen

Analyze & report

  • Worst Offender heat-coloring of high-frequency clash elements
  • Penetrations -- MEP-through-slab detection with size extraction
  • HTML clash reports with screenshots and configurable columns
  • Per-level publishing of NWD/NWF files with clash sidecar data

The plugin runs as a dockable panel with three tabs -- Triage, Analysis, and Settings -- plus ribbon commands and modal dialogs for project setup, publishing, and file tools.

Signing in to ACC / BIM 360

Ribbon > Sign In

Cloud publishing uses an Autodesk Platform Services (APS) app with a Client ID and Client Secret. Enter both once -- in the Sign In dialog, in the Settings tab under Autodesk Cloud Authentication, or in Setup Wizard step 1 -- then sign in from anywhere.

Don't have credentials yet? Someone on your team creates the APS app once at aps.autodesk.com: register a new app (Traditional Web App type), set its callback URL to http://localhost:8080/callback, enable the Data Management, Model Derivative, ACC/BIM 360, and Issues APIs, and share the resulting Client ID and Client Secret with the team. The app must also be added as a custom integration in the ACC/BIM 360 account admin console.
WhereBehavior
Sign In (ribbon)Opens the Sign In dialog: editable Client ID and Client Secret (Show/Hide toggle on the secret), a status dot (green = signed in, red = not), and a Sign In or Sign Out button. Sign In saves the credentials, then opens your default browser to Autodesk's login page and listens on http://localhost:8080/callback for the redirect. Failures show the exact reason in the dialog.
Person icon (Triage quick actions)Live auth indicator to the left of Auto Issue: green person = signed in, grey person with a slash = signed out. Clicking it opens the same Sign In dialog.
Login / Logout (Settings tab)Same flow. A status dot shows green = authenticated, red = not authenticated, gray = checking.
Session persistence. After a successful sign-in the refresh token is stored in %LOCALAPPDATA%\EliteClashManager\auth.json, so you stay signed in across Navisworks restarts. Temporary network failures never log you out -- only a genuinely revoked/expired session does.
If you see...It means / do this
"Client Secret is missing. Please configure it in Settings."Enter the secret in Settings > Autodesk Cloud Authentication (use the Show toggle to verify it).
"Port 8080 is in use by another application..."Another app owns the sign-in callback port -- commonly a dev tool or another add-in. Close it (or its host application) and retry.
"A previous sign-in attempt is still pending. Retry once; if it persists, restart Navisworks."An earlier sign-in was abandoned mid-flow. New attempts automatically take over -- one retry normally clears it.
"Token exchange with Autodesk failed..."The browser step worked but the secret is likely wrong. Verify the Client Secret and check the diagnostic log.

What survives save & reopen

Triage

EliteClashManager stores its durable data as structured comments on each clash group, so the important columns come back automatically when a saved file is reopened -- even on machines running older plugin versions.

DataPersistence
Issue numberPersists. Stored as an "Issue #N" record; the Issue column repopulates on open, and the next cloud sync relinks it live.
Room namePersists. Written by Map Rooms as its own record -- no need to re-run Map Rooms after reopening.
Manager commentsPersist. Notes added via Add Comment are stored per group and shown in the Comment column.
Viewpoints, names, statusesPersist natively in the NWF (viewpoints under Clash Prep; statuses once pushed).
Issue status colorsRe-fetched from the cloud -- click Sync after opening to refresh chips and assignees.
Don't delete the structured comments. In native Navisworks UI these appear as comments like "Issue #465" or "Room: 2-114" on clash groups. They are the plugin's persistent storage -- removing them by hand disconnects the group from its issue or room. Use the plugin's Remove Issue / Remove Comment actions instead.

Auto Issue pipeline

Triage > Auto Issue

One button that runs a full triage cycle -- ideal after refreshing models and re-running tests. The dialog lists each phase with a live status indicator and a scrolling log; press Start and watch it work. When it finishes, the Publish dialog opens with any new groups queued.

Phases 1-5: organize

  • Group by Elevation -- ungrouped clashes get level-band groups
  • Distribute to Existing -- new clashes route to nearby curated groups
  • Absorb into Existing -- clashes inside an existing group's extents join it
  • Split Undistributed -- what's left splits into spatial clusters
  • Sync Cloud Context -- users, types, locations, issue statuses

Phases 6-9: reconcile

  • Close Resolved Issues -- cloud issues whose clashes resolved get closed
  • Clean Up Resolved -- resolved groups and empty containers removed
  • Update Modified Viewpoints -- refreshed for groups that changed
  • Create New Viewpoints -- generated for new groups, redlines baked

Configure per-test grouping beforehand via the gear button in Run Clash Tests (Auto-Group Rules). The pipeline needs cloud sign-in and an active project.

Publish Levels

Ribbon > Tools

Exports the federated model as one file per level -- for field crews and per-floor coordination sets. Models are matched to levels via the filename standard (the level token in each model name).

ControlDetails
Filename PatternProject Name + (Level) + optional Suffix -- a live Output Preview shows the resulting filenames and count.
Export FormatNWD or NWF, with a format-version dropdown.

Consolidate & Rewrite Tests

Ribbon > Tools

Consolidate Appended Files

  • Detects duplicates created by appending NWFs -- clash tests, search sets, and viewpoints suffixed "(filename.nwf)"
  • Shows the detected files and a full action preview grouped into Clash Tests, Saved Viewpoints, and Search Sets

  • The dialog lists every affected clash test that will be rewritten to use them
  • Review the warning text, then Rewrite -- existing results are rebuilt against the new sets

The Room Atlas

Spatial engine

The Room Atlas is the spatial index behind Map Rooms, the room overlay, and location auto-matching. Building it scans the model for Revit rooms, extracts each room's 2D footprint, and computes vertical extents from the project levels (floor to next level, so plenum space above a ceiling still counts as the room below).

TopicDetails
Building itSetup Wizard step 4, or Settings > Rebuild Room Index. Takes up to a minute on large models; results cache per federated file to %LOCALAPPDATA%\EliteClashManager\ECM_Projects\[Project]\[NWF name]\RoomAtlas.json so later sessions load instantly.
When to rebuildAfter room geometry or level structure changes in the Revit models. Day-to-day model refreshes that don't move rooms don't need one.
Blank room resultsThe clash point sits outside every room footprint (shafts, plenums without rooms, exterior), its elevation falls in no level band, or the atlas is stale -- rebuild and re-run Map Rooms. Each Map Rooms run also writes a timestamped RoomMapping_*.log for diagnosis.
Multi-model projectsRooms are matched by physical Z elevation, not level names -- so combined federated models with mismatched level naming still map correctly.
Section 06

Reference

Workflows, troubleshooting, and data locations

Recommended coordination cycle

First session on a project

  • Run the Setup Wizard: sign in, create the project, configure levels + filename pattern + trades, build the Room Atlas
  • Run clash tests, then Group By Level or Group By Elevation
  • Split oversized groups, Map Rooms, rename with [Room]
  • Prep Views, then review each group and refine cameras with Update Viewpoint
  • Issues (N) to publish the first batch

Every model refresh after that

  • Open the file -- project auto-activates, Issue and Room columns restore
  • Click Sync to refresh cloud statuses and assignees
  • Run Auto Issue -- it re-runs grouping, distributes new clashes into your curated groups, closes resolved issues, refreshes viewpoints, and queues anything new for publishing
  • Review the queue, publish, and PUSH any status changes

Troubleshooting

SymptomCause / fix
"No project detected" in the panel headerThe open file matches no project by ACC ID or registered filename. Click the header (or ribbon Active Project), pick or create the project, and leave "Register this file" checked.
"This model is not connected to ACC/BIM 360..."The model lacks cloud linkage -- open it through the ACC/BIM 360 project (so model paths carry nwacc://), not from a plain local copy.
Status changes don't stickSET only stages changes -- orange rows are unsaved. Click PUSH (or press Space), then save the file.
Room column blank for some groupsSee "Blank room results" on the Room Atlas page -- usually a shaft/plenum location or a stale atlas.
Sign-in fails with a port 8080 messageAnother application owns the OAuth callback port. Close the conflicting app (dev tools and other add-ins are the usual suspects) and retry; a "previous attempt still pending" message just needs one retry.
Shortcuts stopped respondingA text box has focus -- all shortcuts (including 1-4) suspend during typing. Click a grid row.
Published pushpins missing in the Coordination Issues add-inAutodesk-side cache: after refreshing models, close and reopen the file to see newly published pushpins.
Navisworks "Out of disk space" on NWD publishUsually a stale Navisworks temp cache, not real disk space -- clear the Navisworks temp folder and retry.
Anything odd -- gather evidenceEnable Diagnostic Logging in Settings, reproduce, then Open Log Folder and check today's EliteClash_YYYYMMDD.log.

Curated groupA group you've refined and given a saved viewpoint -- Distribute treats these as routing targets.
