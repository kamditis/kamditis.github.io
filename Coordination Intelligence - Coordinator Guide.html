Coordinator Guide

Setting up and running the Coordination Intelligence dashboard builder -- from first config to weekly pipeline runs.

What This Tool Does

Coordination Intelligence scans your project files, derives the coordination status of every trade on every level, and generates reports you can share with the GC and trades.

The Pipeline

Step 1
Scan
→
Step 2
Derive State
→
Step 3
Generate Reports

Scan reads your file system (NWC models, overlay DWGs, composites, NWD clash files, signoff PDFs) plus external data (issue Excel, submittal CSV). No interpretation -- just raw facts.

Derive State turns raw facts into named states (Missing, Uploaded, Fresh, Clashed, Clear) by cross-referencing file dates, issue counts, and submittal statuses. Saves the result as project-state.json.

Generate Reports reads the state and produces an Excel workbook (for distribution) and an interactive HTML dashboard (for meetings and review).

Two-phase workflow

Initial setup (once per project): Configure paths, levels, trades, file matching templates, and submittal rules. Takes 30-60 minutes.

Weekly run (before each coordination meeting): Open config, verify level statuses, run pipeline. Takes about 5 minutes.

Launch & Home Page

The app runs as a desktop window. Launch it from the Start Menu, Desktop shortcut, or the executable directly.

When the app opens, you land on the Home page -- a multi-project dashboard showing all your projects at a glance. This is where you add, open, and manage projects.

Config files

Each project is a single JSON config file (e.g., E82.json) that stores all settings: paths, levels, trades, file templates, submittal rules. It does NOT store scan results or state -- those are regenerated each pipeline run. You can copy a config and modify it for a similar project.

Updates

The app checks for updates automatically on startup. If a newer version is available on the shared OneDrive, you'll be prompted to install it.

Severity Badges

Each project card shows which trades have unclashed uploads, color-coded by how long they've been waiting:

  • •Fresh (0-2 days) -- recently uploaded, normal
  • •Ready (3-4 days) -- ready for clashing
  • •Warm (5+ days) -- getting stale, should be clashed soon
  • •Urgent -- meeting day and still unclashed

Day Filter

The left sidebar filters projects by their coordination meeting day. Click a day to see only projects with meetings on that day. The badge counts show how many projects are on each day.

Adding Projects

Add Existing opens a file picker to load an existing JSON config file. Create New starts a blank config. Once a project is in the list, click it to open its config and enter the setup pages (Sources through Actions).

APS Integration

The APS pill in the header shows your Autodesk Platform Services connection status. Click it to authenticate. Once connected, you can:

  • •Link a CI project to its ACC/BIM360 project
  • •Pull coordination issues directly from the API (instead of Excel exports)
  • •Pull design issues separately from the API

Meeting Schedule

Each project can have a meeting schedule configured -- which day(s) of the week the coordination meeting happens. This controls the day filter on the Home page and triggers "Urgent" severity coloring when uploads are waiting on meeting day.

Organize Mode

Click Organize in the header to reorder or remove projects from the list. Cards get dashed borders and drag handles. Click the X on any card to remove it (this doesn't delete the config file, just removes it from the Home page). Click Done when finished.

Card Action Buttons

Each project card has three icon buttons in the top-right corner:

  • •Link icon (chain) -- Link/unlink this project to an Autodesk ACC/BIM360 project, pull issues, refresh metadata
  • •Calendar icon -- Configure the meeting schedule for this project
  • •Document icon -- Generate a report (run the pipeline) for this project without opening it

The link icon is green when linked to Autodesk and gray when unlinked.

Pull vs Scan

Pull (on the Home page) downloads issue data from the Autodesk API into local storage. Scan (on the Issues/Design Team pages) parses whatever data is available -- either the pulled API data or an Excel file, depending on your Sources config. Pull first, then scan (or run the pipeline, which scans automatically).

Quick Add accepts ranges like L00-L37 and comma-separated codes like LR1, LR2. For non-standard codes (like P1-01, P1-RF, MEZZ), type them in directly.

Set each level's status using the dropdown. The status controls how the dashboard treats the level:

  • •Not Started -- coordination hasn't begun yet

Trade Locks

Each level has a Locks button that expands a panel with a checkbox per trade. Check a trade to lock it on that level -- this records a timestamp and means the trade's model is finalized and shouldn't change. If a locked trade's NWC is modified after the lock date, it will be flagged in the dashboard.

When to lock

Lock a trade when its model on that level has been reviewed, all clashes resolved, and the team agrees it's final. This is typically decided during a coordination meeting.

API Issue Filters

If you set the Issue Source to Autodesk API on the Sources page, the API filter panel appears at the top. Check the issue types/subtypes and statuses you want to pull. Save the filters before scanning. This prevents pulling irrelevant issues from ACC.

If using Excel File, this panel is hidden. The tool auto-detects whether the Excel is BIM360 format (has Subtype column) or ACC format (has Category column).

Click Scan Issues to parse. The stats bar shows counts at a glance: Total, Open, In Dispute, No Level, Unknown Trade.

Assignee Mappings

Maps issue assignee names to trades. Expand the section to see the mapping table. Unmapped assignees are flagged with quick-fill buttons so you can add them. Click an unmapped name to prefill the input, select the trade, and click Add.

Trade Abbreviation Overrides

The scanner looks for trade abbreviations in issue titles (e.g., "SM" for Sheet Metal, "PL" for Plumbing). Unmapped abbreviations are flagged the same way. If your project uses non-standard abbreviations, add overrides here.

Filter Bar & Issues Table

Use the filter dropdowns (Level, Trade, Subtype, Status, Assigned To) to drill into specific issues. Check Warnings only to see just the issues with parse problems. The table shows all parsed issues with trade pair resolution, level mapping, and warnings highlighted in amber/red.

Fixing parse warnings

Issues with "No Level" (red) couldn't be mapped to a configured level name. Issues with "Unknown Trade" (amber) have abbreviations not in the lookup table. Fix these by adding assignee mappings or abbreviation overrides, then re-scan to verify they resolve.

The Master-Detail Layout

The left panel is the summary list. It shows every category (NWC, overlays, composites, submittals) grouped by section:

  • •Coordinator -- NWD weekly models, composites, unmatched coordinator files
  • •Per-Trade -- NWC models, overlays (Penetrations, Ceiling Layout, Pads), submittals, unmatched

Each row shows how many files were found and how many are unmatched. Click any row to see the detail on the right.

The right panel shows the level grid. Each level has a dropdown listing ALL files on disk for that trade/type. The scanner pre-fills matches using templates and overrides. You can manually assign any file to any level.

How File Matching Works

The scanner tries to match files in priority order:

  1. Overrides (highest priority) -- manual assignments you make in the dropdowns
  2. Templates -- filename patterns like E82-{LEVEL}-SM.nwc where {LEVEL} matches configured level codes
  3. Auto-detect (lowest priority) -- splits filename on dashes and checks for level codes

Higher-priority matches always win. If a template matches a file to L01, an auto-detect cannot reassign it.

Multi-level files

One file can cover multiple levels. Select the file in the dropdown for each level it covers. For example, an NWD file named "E82-LRF.nwd" might cover both LR1 and LR2. Assign it to both.

Scan All

Click Scan All to run all four scans (NWC, overlays, composites, NWD) in parallel. The left panel updates with fresh file counts. Do this after initial setup and whenever you suspect new files have been uploaded.

Before the Meeting

Your weekly workflow before each coordination meeting -- about 5 minutes.

1Open the app and click into your project

Launch the app. The Home page shows all your projects. Click into the project you want to run. All your settings persist from last time.

2Update level statuses (if needed)

Go to the Levels page. If any levels changed status since last week (a level completed, a new level became active, a level got paused for an RFI), update the dropdowns. Click Apply Changes.

3Update external data files (if needed)

If you have a fresh issue Excel export or submittal CSV, go to Sources and update the file paths. Or if the files are in the same location, they'll be re-read automatically.

4Run the pipeline

Go to the Actions page and click Run Full Pipeline.

During the Run

The status bar at the bottom streams real-time progress. The pipeline runs in the background -- you can browse other pages while it runs. When it finishes, the Open Excel and Open Dashboard buttons activate.

Output Files

The pipeline produces three files in your output directory.

1. project-state.json

The canonical state file. Contains every level, every trade, every derived state. This is what the next pipeline run compares against for drift tracking. You never need to open this file directly.

2. Upload Status Excel

A multi-sheet workbook you can email to trades and the GC. Sheets include:

  • •Overview -- Level x Trade grid of model states, color-coded
  • •Issues Detail -- Open issue counts per level + trade pair breakdowns
  • •Overlay Status -- Per-trade overlay states with submittal info
  • •Composites -- Per-type composite states + For Record dates
  • •Clash Status -- Which trades have fresh models vs stale clash tests
  • •Action Items -- Auto-generated to-do list sorted by priority

3. Interactive HTML Dashboard

A self-contained HTML file you open in a browser. This is the file you share with the GC for their review, and what you use during coordination meetings. It contains all the interactive sections described in the End User Guide.

4. Snapshot (automatic)

A timestamped copy of the state is saved to the snapshot directory. Next time you run the pipeline, you can select this snapshot in the "Compare against" dropdown to see what changed week-over-week.

After the Meeting

Update your config based on what was discussed, then save.

1Update level statuses

If a level was completed or paused during the meeting, go to Levels and update the status. Select a pause reason if applicable.

2Save the config

Click Save in the header bar. The config is ready for next week.

3Distribute reports

Week-over-week tracking

Each pipeline run saves a snapshot. Over time, you build up a history of project state. Select older snapshots in the "Compare against" dropdown to see trends. The dashboard's "Changes" tabs show exactly what moved: which models updated, which overlays were submitted, which issues closed.

Re-export external data

The tool reads your issue Excel and submittal CSV at pipeline time. If you want fresh issue/submittal data in the dashboard, export new files from ACC/BIM360/Procore before running the pipeline. Update the file paths in Sources if the filenames changed.
